## 문서 작성 가이드라인

### 문서의 목적
이 문서들은 **면접 예상질문과 답변 모음이 아닙니다**. 웹 검색을 통해 공식 문서, RFC, 기술 논문 등 신뢰할 수 있는 출처를 기반으로 정리한 **백엔드 로드맵 학습 자료**입니다.

### 작성 원칙
1. **공식 문서 기반**: 각 기술의 공식 문서를 1차 출처로 사용
2. **RFC 참조**: 프로토콜 관련 내용은 반드시 RFC 문서 참조
3. **깊이 있는 설명**: 대기업 2~3년차 수준의 실무 경험을 바탕으로 한 깊이 있는 설명
4. **실용적 예제**: 이론 설명과 함께 실제 동작하는 코드 예제 포함
5. **최신 정보 반영**: 웹 검색을 통해 최신 버전과 트렌드 반영

### 문서 구조
각 README.md 파일은 다음 구조를 따릅니다:
1. 개요 및 핵심 개념
2. 상세 설명 (동작 원리, 아키텍처 등)
3. 코드 예제 및 실습
4. Best Practices
5. 참고 자료 (공식 문서, RFC 링크)

### 참조 출처 우선순위
1. 공식 문서 (Official Documentation)
2. RFC 문서 (IETF)
3. 기술 논문 (ACM, IEEE)
4. 공식 블로그 및 기술 블로그
5. roadmap.sh Backend Roadmap
